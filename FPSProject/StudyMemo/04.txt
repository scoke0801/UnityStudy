유니티 - C#스크립트
- 유니티는 C#언어를 지원한다. 엔진의 코어는 C++로 작성

Start, Update
- 유니티의 가장 기본적인 함수이자 이벤트 함수.
- 이벤트 함수?
>>> 특정 조건을 만족할 떄 시스템(유니티 엔진)에서 자동으로 호출해주는 함수.

유니티 주요 이벤트 함수
Awake
- 스크립트가 실행될 때 한번만 호출되는 함수
- 주로 게임의 상태 값 또는, 변수의 초기화에 사용
- Start함수가 호출되게 전에 맨 먼저 호출
- 스크립트가 비활성화 되어있더라도 실행될
- 코루틴으로 실행 불가

Start
- Update 함수가 호출되기전에 한 번만 호출
- 스크립트가 활성화돼 있어야 실행된다.
- 다른 스크립트의 모든 Awake가 모두 다 실행된 이후에 실행된다.
- 코루틴으로 실행 가능.

Update
- 프레임마다 호출되는 함수로 주로 게임의 핵심 로직을 작성한다.
- 스크립트가 활성화돼 있어야 실행된다.
- 호출 간격이 불규칙적
- 화면의 렌더링 주기와 일치.98

LateUpdate
- 모든 Update함수가 호출되고 나서 한 번씩 호출된다.
- Update함수에서 전처리가 끝난 후 실행되야 하는 로직에 사용한다.
- 스크립트가 활성화돼 있어야 실행된다.

FixedUpdate
- 물리 엔진의 시뮬레이션 계산 주기에 맞춰 실핸된다. 기본값은 0.02초
- 호출되는 주기가 일정하다.

OnEnable
- 게임 오브젝트 또는 스크립트가 활성화됐을 떄 호출
- 주로 사용자 정의 이벤트 연결에 사용자
- 코루틴 사용 불가

OnDisable
- 게임 오브젝트 또는 스크립트가 비활성화 됐을 때 호출된다.
- 이벤트 연결을 종료할 떄 사용한다.
- 코루틴 사용 불가.

OnGUI
- 레거시 GUI 관련 함수를 사용할 때 사용한다.

로컬좌표계와 월드좌표계
- 유니티의 씬 뷰는 무한한 3차원 공간을 표현
- 해당 3차원 공간에서 바뀌지 않는 기준이 되는 좌표를 월드 좌표 또는 글로벌 좌표라 한다.
- 로컬 좌표는 씬 뷰에 존재하는 모든 게임오브젝트의 개별 고유 좌표를 지칭.

접근 제한자
public: 외부 클래스(스크립트)에서 접근 가능
private: 동일 클래스에서만 접근 가능(외부에서는 접근 불가능)
protected: private와 동일하게 외부에서는 접근이 불가능하고, 상속받은 파생클래스에서만 접근 가능.
internal: 같은 어셈블리에서만 접근 가능. 클래스의 경우 접근 제한자를 생락하면 internal이 기본값으로 설정됨.

[ 애니메이션 ]
레거시 애니메이션 
- 하위 호환성을 고려한 애니메이션, 소스 코드로 컨트롤
- Animation 컴포넌트

메카님 애니메이션
- 모션 캡처 애니메이션, 리타게팅 기능
- Animator 컴포넌트

[ 모델 애니메이션 타입 ]
None : 애니메이션을 사용하지 않음.
Legacy : 하위호환성을 유지하기 위한 이전 방식의 애니메이션을
Generic : 메카님 애니메이션. 인체[형 모데링 아닌 3D모델에 적용. 리타게팅할 수 없음.
Humanoid: 메카님 애니메이션. 리타게팅이 가능하며, 사람과 같이 2족 보행하는 모델에 적용.


애니메이션 클립
- 모델의 걷기, 달리기, 점프, 총 쏘기와 같은 동작을 기록한 파일.
- 애니메이션 컴포넌트는 애니메이션 클립에 기록되 관절의 위치와 회전값을 프레임 단위로 재생시키는 역할을 한다.

[그림자]
유니티는 Directional Light, Point Light, SpotLight 광원에 대해 실시간 그림자를 지원.
실시간 그림자는 처리 비용이 비싼 연산.
ShadowType 속성
- No Shadow : 기본 설정값으로 실시간 그림자를 적용하지않는다.
- Hard Shadows : 실시간 그림자를 표현하지만, 외곽선에 계단 현상이 발생한다.
- Soft Shadows : 부드러운 실시간을 그림자를 표현하지만, 가장 많은 부하를 준다.

[ 렌더러 그림자 속성 ]
MeshRenderer or Skinned Mesh Renderer
CastShadow
- off : 그림자를 만들지 않음
- on : 그림자를 만듬
- Two Sided : 백페이스 컬링을 무시하고, 그림자를 양면으로 만듬
- Shadows Only : 그림자를 만들지만, 자신은 렌더링하지 않는다. 즉, 화면에 보이지 않는 특성이 있어 그림자 처리만을 위한 로우 폴리 3D모델에 활용할 수 있다.

RecieveShadows: 다른 그림자에 들어갔을 때 표면에 그림자의 영향을 받는지.

[ 보간 ]
선형 보간
- Linear Interpolation
- 시작점과 끝점 사이의 특정 위치의 값을 추정할 때 사용.
- 현재 값을 목표값으로 변경할 때 갑자기 변경하지 않고, 부드럽게 변경시키는 로직에 많이 사용한다.
- 균일한 속도로 이동시키거나, 회전 시킬 때 사용자

구면 선형 보간
- Spheriacal Linear Interpolation
- 직선의 형태가 아닌 구면(구체) 형태로 값을 추론한다.
- 시작점과 종료점은 느리게 증가하고 중간지점은 동일한 시간 대비 이동해야 할 거리가 크기 떄문에 빠르게 증가.
- 주로 회전 로직에 많이 사용.
